Markdown 转 PDF 工具需求文档

1. 项目概述

开发一款 Python 程序，将包含表格、代码块、PlantUML/Mermaid 图表的 Markdown 技术文档，转换为格式完整、可直接预览图表的单文件 PDF。

2. 核心需求

2.1 输入支持

• 支持单个 .md 或 .markdown 文件作为输入。

• 文档须支持标准 Markdown 语法及以下扩展元素：

    1. 表格
    2. 代码块（需语法高亮）
    3. PlantUML 代码块（用于生成流程图、类图、时序图等）
    4. Mermaid 代码块（用于生成流程图、时序图、甘特图等）

2.2 处理与转换

• 将 Markdown 内容完整、准确地转换为 PDF 格式。

• 代码块处理：

    ◦ 根据标记的语言进行语法高亮。

    ◦ 保留原始格式与缩进。

• 图表渲染：

    ◦ 自动识别 plantuml` 和 ``mermaid` 代码块。

    ◦ 调用相应引擎（PlantUML / Mermaid）将代码块渲染为图片，并嵌入到最终文档中。

• 表格处理：

    ◦ 正确渲染表格的边框、文本对齐方式。

2.3 输出要求

• 输出为单个 PDF 文件。

• 排版清晰，图表与代码显示正常，适用于归档、打印及分发。

3. 可选需求

3.1 导出前预览 (新增)

• 在最终生成和保存 PDF 文件之前，提供预览功能。

• 预览内容：

    1. 显示最终 PDF 的完整版式和页面布局。
    2. 确保所有图表、代码、表格均已正确渲染。
• 预览交互：

    1. 支持缩放和页面滚动。
    2. 能够快速检查分页是否导致图表或表格被不当截断。
• 实现方式建议：可在图形界面中内嵌一个 PDF 预览面板，或在控制台工具中生成临时 PDF 并用系统默认查看器打开。

3.2 样式与配置

• 支持通过 YAML Front Matter 或外部配置文件定义 PDF 元数据（标题、作者）。

• 允许通过自定义 CSS 控制 PDF 基础样式（字体、颜色、边距）。

• 提供命令行界面（CLI），支持基本的输入输出参数。

3.3 扩展处理

• 可配置 PlantUML 服务器地址或本地 JAR 路径。

• 良好的错误处理机制：如图表渲染失败时，在 PDF 中保留原始代码块文本并输出明确警告日志。

4. 非功能性需求

• 准确性：转换结果需与源 Markdown 文档的内容和结构保持一致。

• 性能：处理含多个图表的文档时，响应时间应在可接受范围内。

• 易用性：作为命令行工具时，参数应简洁明了；若具备图形界面，则操作应直观。

5. 技术限制与说明

• 当前仅需支持单文件转换，无需考虑多文件合并或复杂项目结构。

• 程序应优先保证在 Windows 系统下的兼容性与运行便利性。
